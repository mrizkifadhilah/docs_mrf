## ğŸ’¡ Analogi: Surat Tercatat vs. Pesawat Kertas
* **TCP (Transmission Control Protocol) = Surat Tercatat (JNE/Pos Kilat):**
    Kamu mengirim paket. Kamu dapat resi. Kurir memastikan paket sampai ke tangan penerima. Penerima tanda tangan (*Acknowledgement*). Jika paket hilang di jalan, kurir akan mengirim ulang. Paket sampai dengan urutan yang rapi.
    * *Sifat:* Lambat, tapi Dijamin Sampai.

* **UDP (User Datagram Protocol) = Melempar Pesawat Kertas:**
    Kamu melipat 100 pesawat kertas dan melemparnya satu per satu ke temanmu di seberang ruangan secepat mungkin. Mungkin ada 5 pesawat yang jatuh atau tertiup angin. Apakah kamu berhenti untuk memungutnya? Tidak, kamu terus melempar. Yang penting pesannya sampai dengan cepat secara terus-menerus.
    * *Sifat:* Sangat Cepat, tapi Ada yang Hilang (*Packet Loss*).

## ğŸ“š Inti Materi

### 1. Apa itu Socket?
Socket adalah kombinasi dari **IP Address** dan **Port Number**. Bayangkan IP Address sebagai "Alamat Gedung" dan Port sebagai "Nomor Ruangan". Agar aplikasi A bisa bicara dengan aplikasi B, mereka harus membuka "pintu" (socket) ini untuk mengirim dan menerima data.

### 2. TCP: Si Andal (Reliable)
TCP adalah protokol yang berorientasi koneksi (*connection-oriented*).
* **Handshake:** Sebelum data dikirim, TCP melakukan "salaman" 3 tahap (*Three-way Handshake*) untuk memastikan kedua pihak siap.
* **Jaminan:** Data dijamin sampai, utuh, dan berurutan. Jika ada paket data yang hilang, TCP akan meminta pengiriman ulang (*retransmission*).
* **Kegunaan:** Web (HTTP), Email, Chat Text (WhatsApp), Transfer File (FTP). Karena file .zip yang korup 1 bit saja tidak bisa dibuka.

### 3. UDP: Si Cepat (Fast)
UDP adalah protokol tanpa koneksi (*connectionless*).
* **Fire and Forget:** Data dikirim begitu saja tanpa "salaman" awal.
* **Tanpa Jaminan:** Tidak ada garansi data sampai. Tidak ada pengiriman ulang. Paket data bisa sampai secara acak (tidak berurutan).
* **Kegunaan:** Game Online (FPS/MOBA), Video Streaming, Voice Call (VoIP).
* **Mengapa?** Dalam game tembak-menembak, data posisi musuh 1 detik yang lalu sudah "basi". Tidak ada gunanya mengirim ulang data lama. Lebih baik fokus mengirim data posisi *sekarang* secepat mungkin.

## ğŸ“± Contoh Penerapan: Mobile Game (PUBG/Mobile Legends)
1.  **Saat Login & Update (TCP):** Menggunakan TCP. Kamu tidak mau *skin* yang kamu beli gagal masuk ke inventaris karena paket datanya hilang. Data akun harus 100% akurat.
2.  **Saat Pertandingan (UDP):** Menggunakan UDP. Server membombardir HP-mu dengan koordinat posisi 10 pemain lain setiap milidetik. Jika satu paket koordinat hilang (membuat karakter musuh terlihat "teleport" sedikit), itu lebih baik daripada game-nya *pause* menunggu data itu dikirim ulang.

## ğŸ“– Mini-Glossary
* **Socket:** Titik akhir komunikasi jaringan (IP + Port).
* **Handshake:** Proses awal pembentukan koneksi di TCP (Syn, Syn-Ack, Ack).
* **Packet Loss:** Kondisi ketika data yang dikirim hilang di tengah jalan (wajar di UDP).
* **Latency:** Waktu tunda yang dibutuhkan data untuk sampai dari sumber ke tujuan.
